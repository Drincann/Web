    pyinstaller 语法

    pyinstaller [options] name.py

    常用的可选参数如下：
    --onefile 将结果打包成一个可执行文件
    --onedir 将所有结果打包到一个文件夹中，该文件夹包括一个可执行文件和可执行文件执行时需要的依赖文件（默认）
    --paths=DIR 设置导入路径
    --distpath=DIR 设置将打包的结果文件放置的路径
    --specpath=DIR 设置将spec文件放置的路径
    --windowed 使用windows子系统执行，不会打开命令行（只对windows有效）
    --nowindowed 使用控制台子系统执行（默认）（只对windows有效）
    --icon= 将file.ico添加为可执行文件的资源(只对windows有效）

    如pyinstaller --paths="D:\Queena" guess_exe.py

    示例: pyinstaller -F wxy_tool.py --distpath=wxy-mtt

Git

    • 1. origin 克隆到本地
      git clone [https://github.com/xxx/xxx / git@github.com:xxx/xxx]
    • 2. 创建分支
      git checkout [branch-name]
    • 3. 修改代码
    • 4. commit 到本地仓库
      git add [. / file_name]
      git commit -m "description"
    • 5. 将 upstream 同步到本地
      git upstream [branch-name(master)]
      git rebase upstream/[branch-name(master)]
    • 6. 本地 branch 推送到 origin
      git push -f origin [branch-name]
      这里的 -f 意为强覆盖，以强覆盖方式替代 origin 内容
    • 7.1 pr 后删除远程分支
      git push origin --delete [branch-name]
      7.2 切换到主分支
      git checkout master -f
      7.3 删除本地分支
      git branch -D [branch-mame]
    • 8. 用 upstream 版本来更新本地(主)分支
      git pull --ff upstream [branch-name(master)]
    • 文件操作
      ls file_name 列出清单
      cat file_name 查看和修改文件内容
      (git) rm file_name 删除文件
      git checkout -- file_name 用版本库里的版本替换工作区版本(恢复文件
      cp old_file_name new_file_name 复制文件
    • 查看分支情况
      git branch
    • 开发环境信息
      git remote 检查远程仓库信息
      git remote add upstream/origin [url] 添加远程仓库信息
